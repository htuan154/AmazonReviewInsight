9) Runbook tái lập

Chạy full (1 lệnh/phase)

# ETL (Ngày 1) — ví dụ đường dẫn
spark-submit code_v2/etl/preprocess_spark_v2.py \
  --in hdfs://localhost:9000/amazon_v2/raw \
  --out hdfs://localhost:9000/output_v2/cleaned

# Features (Ngày 2) — v3 khuyến nghị
python code_v2/features/feature_pipeline_v2.py \
  --data hdfs://localhost:9000/output_v2/train \
  --out hdfs://localhost:9000/output_v2/features_train_v3 \
  --feature_set v3 --include_text --include_sentiment

# Train (Ngày 3) — LightGBM
python code_v2/models/train_lightgbm_v2_hdfs.py \
  --train webhdfs://localhost:9870/output_v2/features_train_v3 \
  --test  webhdfs://localhost:9870/output_v2/features_test_v3 \
  --out   ./output_v2/lightgbm_v2_full_hdfs \
  --val_ratio 0.1 --threads 8 --boost_rounds 3000 \
  --early_stopping_rounds 200 --learning_rate 0.035 \
  --num_leaves 128 --feature_fraction 0.8 --bagging_fraction 0.8 \
  --bagging_freq 2 --min_child_samples 60 --reg_alpha 0.1 --reg_lambda 0.2

# Predict & Submit (Ngày 7)
python code_v2/models/predict_pipeline_v2.py \
  --test amazon_v2/test \
  --model output_v2/lightgbm_v2_full_hdfs/model.txt \
  --features output_v2/lightgbm_v2_full_hdfs/run_manifest.json \
  --out output_v2/submission_tmp.csv --mode proba --batch_size 200000

python code_v2/utils/clean_submission.py \
  --input output_v2/submission_tmp.csv \
  --output output_v2/submission_final.csv --strategy first

Phụ lục B — Chính sách xử lý NULL (chi tiết)

  • Tự động phát hiện cột có NULL; áp dụng chiến lược theo kiểu dữ liệu.
  • Numeric: ưu tiên median/mean theo category; fallback median/mean toàn cục; cuối cùng giá trị mặc định (0 hoặc 3.0 cho rating TB).
  • String: thay thế “Unknown”/mode; bổ sung cột chỉ báo has_* khi cần.
  • Kiểm định: đếm NULL trước/sau; log thống kê; dừng nếu còn NULL trong feature số/nhị phân.
# Ví dụ impute price theo chuỗi fallback
price_imputed = coalesce(price, price_cat_median, global_median, 0.0)
