PHẦN 2: LIGHTGBM TRAINING

2.1. Tại sao dùng LightGBM?

Hạn chế của Logistic Regression:

  • Linear model: Chỉ nắm bắt được mối quan hệ tuyến tính giữa features
  • Không tương tác: Không tự động học được tương tác giữa các features (length × sentiment, rating × price)
  • Decision boundary đơn giản: Chỉ là đường/mặt phẳng phân tách

Ưu điểm của LightGBM:

  • Tree-based model: Tự động học được non-linear patterns
  • Feature interactions: Tự động nắm bắt tương tác phức tạp (VD: review dài + sentiment tiêu cực = rất hữu ích)
  • Handle imbalance tốt: Tham số isUnbalance=True tối ưu cho dữ liệu mất cân bằng
  • Feature importance: Cho biết feature nào quan trọng nhất
  • Robust to outliers: Tree splits ít bị ảnh hưởng bởi giá trị ngoại lai

2.2. LightGBM Configuration

Tham số Giá trị Giải thích
objective binary Bài toán phân loại nhị phân (helpful/not helpful)
numLeaves 31 Số lá tối đa mỗi cây (default, cân bằng tốt)
numIterations 100 Số cây decision tree (100 trees = ensemble mạnh)
learningRate 0.1 Tốc độ học (moderate, tránh overfitting)
featureFraction 0.8 Dùng 80% features mỗi tree (tăng diversity)
baggingFraction 0.8 Dùng 80% data mỗi iteration (giảm overfitting)
baggingFreq 5 Bagging mỗi 5 iterations
isUnbalance True Xử lý class imbalance (24.8% vs 75.2%)

PHẦN 4: KEY TAKEAWAYS & INSIGHTS

1. LEAN Grid Strategy rất hiệu quả

  • Chỉ cần 8 configs (vs 108) để đạt ~90% accuracy của full grid
  • Thời gian: 8.5 phút (vs 10-16 giờ) - 75x nhanh hơn!
  • Focus vào extremes (0.0 vs 0.01) thay vì test tất cả intermediate values
  • Trade-off xứng đáng cho dự án có thời hạn chặt

2. LightGBM >> Logistic Regression

  • AUC-PR: 0.6548 vs 0.5441 (+20.3% improvement)
  • Lý do: Tree-based models nắm bắt được non-linear patterns và feature interactions
  • Ví dụ pattern: Review dài (200+ words) + sentiment tiêu cực + 1-2 sao = RẤT hữu ích
  • LogReg không thể học pattern này vì chỉ là tổ hợp tuyến tính

3. Feature Importance khớp với Analysis

  • Top 1: product_total_ratings (306) - Sản phẩm phổ biến có nhiều review chất lượng
  • Top 2: review_length_log (201) - Xác nhận "Length is King" từ Day 5 analysis!
  • Top 3: star_rating (133) - Review phê bình (1-2 sao) hữu ích hơn 5 sao
  • Khớp hoàn toàn với 10 insights từ feature_analysis.json!

4. Regularization và Noise Filtering

  • regParam=0.01: L2 regularization rất quan trọng (0.52+ vs 0.41-0.45 không reg)
  • minDocFreq=10: Lọc noise tốt hơn minDocFreq=5
  • 20K features đủ: 50K không tăng performance, chỉ tăng complexity
  • High-dimensional text data cần strong regularization để chống overfitting
